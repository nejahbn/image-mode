== Bootable Containers

Bootable containers o **bootc** es una herramienta que se utiliza para crear contenedores que incluyen el kernel, el cargador de arranque u otros componentes que no se incluyen en los contenedores normalmente. Una vez creado un contenedor de este tipo, se sube a un registry, al igual que cualquier contenedor y mediante **image builder** podemos crear imágnenes de RHEL que se desplegarán sobre cualquier infraestructura, máquinas físicas, virtuales o en proveedores cloud.

image::containervsbootc.png[]
       
Esta nueva forma de gestionar el sistema operativo también permite actualizar dichos componentes a partir de imágenes de contenedores bootables.

image::bootcnutshell.png[]

Bootable containers son contenedores que implementan el estándar de OCI para construir un contenedor que incluye el sistema operativo junto con el kernel. Todo esto se hace utilizando las herramientas ya existentes para construir contenedores, como https://www.redhat.com/en/topics/containers/what-is-podman[podman].

[IMPORTANT]
----
RHEL Image Mode se encuentra actualmente disponible en RHEL 9.4 como Technology Preview (TP), lo que significa que no esta listo para su uso en producción pero puede usarse para ir familiarizándose con la tecnología para que cuando pase a General Availability (GA) los clientes que lo consideren útil puedan empezar a usarlo desde el primer momento al ya conocer la tecnología.
